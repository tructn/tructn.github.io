- `ValueTask<T>` for efficient async APIs

## Key Concepts & Implementation

### 1. ValueTask&lt;T&gt; for Efficient Async APIs

The `ILookupService` interface returns a `ValueTask<IList<LookupModel>>`, which is more efficient than `Task<T>` for high-throughput scenarios when results are often cached or synchronous.

```csharp
internal interface ILookupService
{
	ValueTask<IList<LookupModel>> GetCategoriesAsync();
}
```

> **Tip:** Use `ValueTask<T>` when the result is often available synchronously (e.g., cache). This avoids unnecessary allocations and improves performance, especially under high load.

### 2. Decorator Pattern with Scrutor

Scrutor enables easy registration of decorators in DI. Here, `LookupCacheDecorator` wraps the actual `LookupService` to add caching logic:

```csharp
builder.Services.AddScoped<ILookupService, LookupService>();
builder.Services.Decorate<ILookupService, LookupCacheDecorator>();
```

The decorator checks the cache before calling the inner service:

```csharp
public async ValueTask<IList<LookupModel>> GetCategoriesAsync()
{
	var isCacheMissed = false;
	var categories = await _cache.GetOrCreateAsync("/categories", async (ct) =>
	{
		isCacheMissed = true;
		_logger.LogInformation("[CACHE MISSED] Getting categories from normal lookup service");
		var categories = await _innerLookupService.GetCategoriesAsync();
		return categories;
	});

	if (!isCacheMissed)
	{
		_logger.LogInformation("[CACHE HIT] Categories were found in cache");
	}

	return categories;
}
```

### 3. Hybrid Cache with Redis Backplane


The hybrid cache combines in-memory and distributed (Redis) caching for optimal performance and scalability. 

```csharp
builder.Services.AddHybridCache(options =>
{
	options.DefaultEntryOptions = new HybridCacheEntryOptions
	{
		Expiration = TimeSpan.FromMinutes(5),
		LocalCacheExpiration = TimeSpan.FromMinutes(5)
	};
});
builder.Services.AddStackExchangeRedisCache(options =>
{
	options.Configuration = "localhost:6379";
	options.InstanceName = "hybrid-cache-demo";
});
```

This means cache entries will expire from both memory and Redis after 5 minutes by default.

## Sample Endpoint

```csharp
app.MapGet("/v1/categories", async (ILookupService service) =>
```

## Pros and Cons

### Pros

- **Performance:** Hybrid cache provides fast in-memory reads and distributed consistency.
- **Scalability:** Redis backplane enables cache sharing across instances.
- **Separation of Concerns:** Decorator pattern keeps caching logic separate from business logic.
- **Efficiency:** `ValueTask<T>` reduces allocations for frequently cached results.

### Cons

- **Complexity:** More moving parts (decorators, hybrid cache, Redis) than a simple cache.
- **Redis Dependency:** Requires Redis server for distributed cache.
- **Cache Invalidation:** Must design cache expiration and invalidation carefully. Not covered in this demo for simplicity.
- **Potential Data Inconsistency:** With multiple servers behind a load balancer, each server checks its own memory cache first. This means data may be briefly inconsistent between servers until the cache expires or is invalidated.
